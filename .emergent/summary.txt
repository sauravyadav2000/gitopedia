<analysis>**original_problem_statement:**
Build a full-stack web application called Gitopedia — a GitHub repository intelligence tool. Users input a GitHub repo URL, and the system uses an LLM (Claude) to generate a structured Markdown report. Reports are public and free to read, while generating and editing them costs credits. The application must support a credit-based economy with Stripe integration for purchasing credits.

**PRODUCT REQUIREMENTS:**
- **Core Functionality:** Generate in-depth reports on public GitHub repositories.
- **Credit System:**
    - 3 free credits on signup.
    - 2 credits to generate a report.
    - 1 credit to edit and republish.
    - Reading reports is free.
- **Monetization:** Stripe integration to buy credits ( for 5,  for 15, 0 for 35).
- **Report Quality:** The output must be high-quality, including a directory tree map, Mermaid diagrams for service communication, and ER diagrams for database migrations. The output should be rich HTML/React, not plain Markdown.
- **User Management:** Firebase for authentication and user management.
- **Payment History:** Users should be able to view their transaction history.
- **Technical Constraints:**
    - Deduplicate report generation for the same repository.
    - Cache GitHub API data.
    - Handle large files gracefully.
    - Stream LLM responses to the frontend using Server-Sent Events (SSE).
    - Ensure credit transactions are atomic and refund credits on generation failure.
    - Implement a fallback model for the LLM.
